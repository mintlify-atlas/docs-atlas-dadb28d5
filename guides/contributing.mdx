## Getting Started

OpenPencil is open source and welcomes contributions. This guide will help you set up your development environment and understand the contribution workflow.

## Setup

### Prerequisites

- [Bun](https://bun.sh/) ‚Äî JavaScript runtime and package manager
- [Git](https://git-scm.com/) ‚Äî Version control
- [Rust](https://rustup.rs/) ‚Äî Only needed for desktop app development

### Clone and Install

```bash
git clone https://github.com/open-pencil/open-pencil.git
cd open-pencil
bun install
```

## Development Workflow

### Running the App

**Web version (recommended for quick iteration):**

```bash
bun run dev
```

This starts a Vite dev server at `http://localhost:1420` with hot reload.

**Desktop app:**

```bash
bun run tauri dev
```

Builds and runs the Tauri desktop app with hot reload.

### Before Submitting a PR

Run all quality checks to ensure your code meets project standards:

```bash
# Lint and typecheck
bun run check

# Format code
bun run format

# Check for code duplication (must be < 3%)
bun run test:dupes

# Run unit tests
bun test tests/engine/

# Run end-to-end tests
bun run test
```

## Project Structure

OpenPencil uses a monorepo structure with Bun workspaces:

### Packages

- **`packages/core`** ‚Äî `@open-pencil/core`
  - Scene graph, renderer, layout, codec
  - Zero DOM dependencies, runs headless in Bun
  - Core functionality used by all other packages

- **`packages/cli`** ‚Äî `@open-pencil/cli`
  - Headless CLI for `.fig` inspection and export
  - Uses `citty` for CLI framework and `agentfmt` for output formatting

- **`packages/mcp`** ‚Äî `@open-pencil/mcp`
  - MCP server for AI coding tools
  - Supports both stdio and HTTP transports
  - Reuses tools from core package

### Application Code

- **`src/`** ‚Äî Main Tauri/Vite desktop editor
  - `src/ai/` ‚Äî AI tool wiring
  - `src/components/` ‚Äî Vue components
  - `src/composables/` ‚Äî Vue composables for canvas, keyboard, collaboration
  - `src/stores/` ‚Äî Editor state management
  - `src/engine/` ‚Äî Re-export shims from `@open-pencil/core`

- **`desktop/`** ‚Äî Tauri v2 configuration and Rust code

- **`tests/`**
  - `tests/e2e/` ‚Äî Playwright visual regression tests
  - `tests/engine/` ‚Äî Unit tests
  - `tests/fixtures/` ‚Äî Test `.fig` files (Git LFS)

## Code Conventions

### General Rules

- **Bun runtime** ‚Äî Use Bun, not Node.js
- **TypeScript** ‚Äî All code is TypeScript
- **No `any`** ‚Äî Use proper types, generics, or unknowns
- **No `!` assertions** ‚Äî Use guards, optional chaining (`?.`), or nullish coalescing (`??`)
- **No `Math.random()`** ‚Äî Use `crypto.getRandomValues()` for all randomness

### Imports

- **`@/` alias** ‚Äî Use for app-level cross-directory imports (e.g., `@/stores/editor`)
- **Relative imports** ‚Äî Use within `packages/core` (e.g., `../../types`)

### Styling

- **Tailwind 4** ‚Äî All styling via Tailwind classes
- **No inline CSS** ‚Äî Avoid `style` attributes
- **No `<style>` blocks** ‚Äî Don't use component-scoped styles

### Icons

- **unplugin-icons** ‚Äî Use with Iconify/Lucide collections
- **Component syntax** ‚Äî `<icon-lucide-menu />` (not raw SVG or Unicode)

### Vue

- **Vue 3 Composition API** ‚Äî Use `<script setup>`
- **VueUse** ‚Äî Use `@vueuse/core` hooks (e.g., `useEventListener`)
- **Reka UI** ‚Äî Use for UI components (Splitter, ContextMenu, etc.)

### Constants

- **`src/constants.ts`** ‚Äî Define constants here, not inline magic numbers

## Browser API Guards

Code in `packages/core` must guard browser APIs since it runs headless in Bun:

```typescript
if (typeof window !== 'undefined') {
  // Browser-only code
}

if (typeof document === 'undefined') {
  // Headless-only code
}
```

## Test Fixtures

Test `.fig` files in `tests/fixtures/` use Git LFS:

- **Regular push** ‚Äî `git push` (slow, uploads LFS files)
- **Skip LFS** ‚Äî `git push --no-verify` (fast, skips LFS pre-push hook)

**Rule:** Only use regular `git push` when you've changed `.fig` fixtures.

## Documentation

Update relevant docs when making changes:

- **`CHANGELOG.md`** ‚Äî Add user-facing changes under "Unreleased"
- **`README.md`** ‚Äî Update if feature descriptions change
- **`AGENTS.md`** ‚Äî Update if architecture or conventions change
- **`packages/docs/`** ‚Äî Update VitePress docs for major features

## Commit Messages

Follow the existing style in `git log`:

- Keep messages concise
- Use imperative mood ("Add feature" not "Added feature")
- Reference issues when applicable

## CLI Development

### Output Formatting

- **Always use `agentfmt`** ‚Äî All CLI output must use formatters from `agentfmt`
- **Available formatters** ‚Äî `fmtList`, `fmtHistogram`, `fmtSummary`, `fmtTree`, `kv`, `entity`, `bold`, `dim`
- **Don't hand-roll formatting** ‚Äî Use helpers from `packages/cli/src/format.ts`
- **JSON flag** ‚Äî Every command supports `--json` for machine-readable output

### Commands

```bash
bun open-pencil info <file>      # Document stats
bun open-pencil tree <file>      # Node tree
bun open-pencil find <file>      # Search nodes
bun open-pencil export <file>    # Render to PNG/JPG/WEBP
```

## Tool Development

Tools (AI / MCP / CLI operations) are defined once in `packages/core/src/tools/schema.ts`:

1. Add a `defineTool()` in `schema.ts`
2. Add to `ALL_TOOLS` array
3. Tool is instantly available in:
   - AI chat (via Vercel AI adapter)
   - MCP server (stdio + HTTP)
   - CLI `eval` command

### Tool Structure

```typescript
const myTool = defineTool({
  name: 'my_tool',
  description: 'What this tool does',
  parameters: v.object({
    param: v.string(),
  }),
  execute: async (figma, args) => {
    // Implementation
  },
})
```

## Rendering

- **Canvas** ‚Äî CanvasKit (Skia WASM) on WebGL, not DOM
- **Versioning** ‚Äî `sceneVersion` for scene mutations, `renderVersion` for repaints
- **Render requests** ‚Äî `requestRender()` bumps both, `requestRepaint()` only bumps render
- **Culling** ‚Äî Off-screen nodes are skipped, but unclipped parents aren't

## Collaboration

- **P2P** ‚Äî Trystero (WebRTC), no server
- **CRDT** ‚Äî Yjs for document state
- **Persistence** ‚Äî y-indexeddb for local storage
- **Composable** ‚Äî `src/composables/use-collab.ts`

## File Format

- **`.fig` files** ‚Äî Kiwi binary codec, same as Figma
- **Schema** ‚Äî `packages/core/src/kiwi/codec.ts`
- **Vector data** ‚Äî Reverse-engineered `vectorNetworkBlob` format
- **Testing** ‚Äî Round-trip by exporting and reimporting in Figma

## Desktop App

- **Tauri v2** ‚Äî Rust backend, web frontend
- **Plugins** ‚Äî dialog, fs, opener
- **Permissions** ‚Äî Configure in `desktop/tauri.conf.json`

## Release Process

Releases are handled by maintainers:

1. Update versions in `package.json`, `packages/*/package.json`, `desktop/tauri.conf.json`
2. Update `CHANGELOG.md`
3. Commit: `Release v0.x.y`
4. Tag: `git tag v0.x.y && git push --tags`
5. CI builds binaries and publishes to npm

## CI Workflows

| Workflow | Trigger | Action |
|----------|---------|--------|
| `build.yml` | `v*` tag | Build desktop apps, publish to npm |
| `app.yml` | Push to `master` | Deploy web app to Cloudflare Pages |
| `docs.yml` | `packages/docs/**` | Deploy docs to Cloudflare Pages |
